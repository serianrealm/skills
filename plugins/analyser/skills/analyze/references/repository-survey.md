# Repository Survey

Collect facts before interpretation:

- repository root, branch, commit, dirty state
- worktrees and branches
- languages, package managers, build systems
- test runners, linters, formatters, type checkers
- CI, Docker, systemd, deployment files
- CLI and runtime entry points
- Claude Code agents, skills, hooks, MCP files, project instructions
- config files and environment variable names
- test directories and naming conventions

Skip secret-bearing files: `.env`, credentials, private keys, token stores, and files with obvious secret extensions. Record environment variable names only.
